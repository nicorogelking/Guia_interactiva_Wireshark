El Origen de la Comunicación: ¿Por qué OSI?

A finales de los años 70, la computación vivía en una "Torre de Babel". Empresas gigantes como IBM y las primeras máquinas de Mac tenían un problema crítico: eran sistemas cerrados. Si comprabas una máquina IBM, esta solo podía conectarse a otra IBM usando un cable y protocolo específico de IBM.

Esto era un desastre para cualquier empresa que quisiera crecer o mezclar tecnologías. Para solucionar esto, nació el Modelo OSI (Open Systems Interconnection), basado en tres pilares:

  • Estandarización: Un lenguaje común para todos.
  • Independencia: Cada capa hace su trabajo sin molestar a la otra.
  • Dividir y Conquistar: Fragmentar la complejidad en 7 pasos simples.

La Analogía del Océano: Imagina que el Internet es un océano por el que enviamos un paquete.

• Capas 7 a 5: Preparan el mensaje (el contenido del paquete y cómo se ve).
• Capa 4: Es el guardaespaldas. Se asegura de que el paquete llegue completo y sin daños al otro lado.
• Capas 3 a 1: Son el transporte físico (el barco, el motor y las rutas marinas).

1. Guía de Instalación de Wireshark

Ahora que entendemos cómo viajan los datos por las capas, usemos Wireshark para "ver" esos paquetes en tiempo real.

🪟 Windows

  1. Ve a wireshark.org y descarga el instalador de 64-bit.
  2. Inicia la instalación y presiona "Siguiente".
  3. ¡IMPORTANTE! Instala Npcap cuando se te solicite; es el motor que captura los paquetes.

🍎 macOS

  1. Descarga el archivo .dmg desde la página oficial.
  2. Instala arrastrando a Aplicaciones.
  3. O usa Homebrew: brew install --cask wireshark.

🐧 Linux (Ubuntu/Debian)

  1. sudo apt update && sudo apt install wireshark
  2. Selecciona Sí cuando pregunte por permisos de usuario no-root.
  3. Ejecuta: sudo usermod -aG wireshark $USER y reinicia sesión.

2. Desafíos de la Ayudantía

Aplica lo aprendido sobre el Modelo OSI resolviendo estos casos reales.

Nivel 1: Básico

Desafío 1: El "Ping-Pong" (ICMP)

Tarea: Captura el tráfico de un ping 8.8.8.8 y filtra por icmp. Observa cómo se ve una petición de Capa 3.

Solución: Filtra por icmp. Verás el Echo request y el Echo reply.
Nivel 2: Intermedio

Desafío 2: El Saludo de 3 Vías (TCP)

Tarea: Entra a http://neverssl.com y busca el saludo TCP (SYN, SYN-ACK, ACK). Este es el trabajo de la Capa 4.

Solución: Haz clic derecho en un paquete HTTP -> Follow -> TCP Stream. Verás los flags de conexión en los primeros paquetes.
Nivel 3: Avanzado

Desafío 3: Segmentación de Datos

Tarea: Descarga un archivo pesado y observa cómo el "guardaespaldas" (TCP) fragmenta el archivo en múltiples paquetes para que quepan en la red.

Solución: Verás muchos paquetes seguidos que dicen [TCP segment of a reassembled PDU].
