El Origen de la Comunicación: ¿Por qué OSI?

A finales de los años 70, la computación vivía en una "Torre de Babel". Empresas gigantes como IBM y las primeras máquinas de Mac tenían un problema crítico: eran sistemas cerrados. Si comprabas una máquina IBM, esta solo podía conectarse a otra IBM usando un cable y protocolo específico de IBM.

Esto impedía crear redes empresariales diversas. Para solucionar esto, nació el Modelo OSI (Open Systems Interconnection), basado en tres pilares:

  • Estandarización: Un lenguaje común para todos.
  • Independencia: Cada capa hace su trabajo sin molestar a la otra.
  • Dividir y Conquistar: Fragmentar la complejidad en 7 pasos simples.

La Analogía del Océano: Imagina que el Internet es un océano por el que enviamos un paquete.

• Capas 7 a 5: Preparan el mensaje (el contenido del paquete y cómo se ve).
• Capa 4: Es el guardaespaldas. Se asegura de que el paquete llegue completo y sin daños al otro lado.
• Capas 3 a 1: Son el transporte físico (el barco, el motor y las rutas marinas).

1. Guía de Instalación de Wireshark

Usemos Wireshark para observar estos paquetes viajando a través de las capas en tiempo real.

🪟 Windows

  1. Descarga el instalador en wireshark.org.
  2. Inicia la instalación y presiona "Siguiente".
  3. ¡IMPORTANTE! Instala Npcap; sin esto no podrás capturar tráfico.

🍎 macOS

  1. Descarga el archivo .dmg e instala arrastrando a Aplicaciones.
  2. Alternativa: brew install --cask wireshark.

🐧 Linux (Ubuntu/Debian)

  1. sudo apt update && sudo apt install wireshark
  2. Selecciona Sí para permitir capturas a usuarios no-root.
  3. Ejecuta: sudo usermod -aG wireshark $USER y reinicia sesión.

2. Desafíos de la Ayudantía

Aplica lo aprendido sobre el Modelo OSI resolviendo estos casos reales.

Nivel 1: Básico

Desafío 1: El "Ping-Pong" (ICMP)

Tarea: Captura el tráfico de un ping 8.8.8.8 y filtra por icmp. Observa la comunicación en Capa 3.

Solución: Filtra por icmp. Verás el Echo request (salida) y el Echo reply (entrada).
Nivel 2: Intermedio

Desafío 2: El Saludo de 3 Vías (TCP)

Tarea: Entra a http://neverssl.com y busca el saludo TCP (SYN, SYN-ACK, ACK). ¡Este es el guardaespaldas de la Capa 4 en acción!

Solución: Clic derecho en un paquete HTTP -> Follow -> TCP Stream. Observa los flags SYN y ACK al inicio de la conexión.
Nivel 3: Avanzado

Desafío 3: Segmentación de Datos

Tarea: Descarga un archivo y observa cómo el protocolo de transporte divide el contenido en múltiples fragmentos para que puedan viajar por el medio físico.

Solución: Verás paquetes con la etiqueta [TCP segment of a reassembled PDU].
