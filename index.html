Taller Práctico: Análisis de Redes con Wireshark 🦈

Guía de instalación y desafíos prácticos paso a paso

1. Guía de Instalación

Antes de comenzar a capturar tráfico, necesitamos tener la herramienta instalada. Sigue las instrucciones según tu sistema operativo:

🪟 Windows

  1. Ve a la página oficial wireshark.org y descarga el instalador de 64-bit.
  2. Inicia la instalación y presiona "Siguiente".
  3. ¡IMPORTANTE! Cuando te pregunte si deseas instalar Npcap, asegúrate de marcar la casilla. Esto es vital para capturar el tráfico de tu tarjeta de red.

🍎 macOS

  1. Descarga el archivo .dmg (Intel o Apple Silicon según tu Mac) desde la página oficial.
  2. Abre el archivo e instala arrastrando a Aplicaciones.
  3. Alternativa para desarrolladores: Si usas Homebrew, abre la terminal y ejecuta brew install --cask wireshark.

🐧 Linux (Ubuntu/Debian)

  1. Abre tu terminal.
  2. Ejecuta: sudo apt update && sudo apt install wireshark
  3. Cuando pregunte si los usuarios no-root pueden capturar paquetes, selecciona Sí (Yes).
  4. Agrega tu usuario al grupo: sudo usermod -aG wireshark $USER y reinicia tu sesión.

2. Desafíos de la Ayudantía

Aplica lo aprendido. Ve subiendo de nivel resolviendo estos casos de uso reales.

Nivel 1: Básico

Desafío 1: El "Ping-Pong" (ICMP)

Objetivo: Aprender a seleccionar la interfaz de red, capturar tráfico y usar filtros simples.

Tarea:

  1. Abre Wireshark y haz doble clic en tu interfaz activa (la que tiene gráficos moviéndose, como Wi-Fi o Ethernet).
  2. Abre una terminal/consola en tu computador y ejecuta el comando: ping 8.8.8.8 (déjalo dar unos 4 saltos y detenlo).
  3. Vuelve a Wireshark y detén la captura (botón cuadrado rojo).
  4. Aplica un filtro para ver solo este tráfico y encuentra la petición y la respuesta.

Solución:
En la barra superior verde de Wireshark, escribe icmp y presiona Enter.
Verás paquetes clasificados como Echo (ping) request (tú preguntando) y Echo (ping) reply (Google respondiendo). Este es el protocolo que verifica si hay conectividad.
Nivel 2: Intermedio

Desafío 2: El Saludo de 3 Vías y "Follow Stream"

Objetivo: Identificar cómo se inicia una conexión TCP y aislar una única comunicación.

Tarea:

  1. Inicia una nueva captura en Wireshark (sin guardar la anterior).
  2. Abre tu navegador y entra a un sitio sin cifrar, por ejemplo: http://neverssl.com
  3. Detén la captura.
  4. Encuentra el inicio de esa conexión (El saludo de 3 partes: SYN, SYN-ACK, ACK) y aísla esa conversación para ver los mensajes crudos entre tu PC y la web.

Solución:
1. En la barra de filtros escribe http y presiona Enter.
2. Busca el paquete que diga "GET / HTTP/1.1", hazle clic derecho -> Follow -> TCP Stream.
3. Se abrirá una ventana con texto rojo (lo que tú enviaste) y azul (lo que el servidor respondió).
4. Cierra la ventana. Verás que el filtro cambió a algo como tcp.stream eq 0. Si miras los 3 primeros paquetes de esa lista, verás el famoso saludo: [SYN], luego [SYN, ACK], y finalmente [ACK].
Nivel 3: Avanzado

Desafío 3: El elefante en la red (Paquetes Cortos vs Largos)

Objetivo: Comprender la segmentación TCP combinando filtros complejos.

Tarea:

  1. Inicia una nueva captura.
  2. Busca en Google una imagen grande (o un archivo PDF de varios megas) y descárgala.
  3. Detén la captura.
  4. Encuentra esa descarga utilizando una combinación de filtros de IP y Protocolo. Identifica visualmente por qué es un "TCP largo" (segmentado) en lugar de una petición corta.

Solución:
1. Filtra por tráfico seguro o web: tcp.port == 443 or tcp.port == 80.
2. Busca un flujo que tenga mucha densidad de paquetes de un servidor a tu IP.
3. Haz Follow TCP Stream. Al cerrar la ventana, notarás cientos de paquetes consecutivos que dicen [TCP segment of a reassembled PDU].
Explicación: Los paquetes TCP tienen un tamaño máximo (MTU, usualmente 1500 bytes). Un paquete corto (ej. un ping o un simple "GET") cabe en 1 paquete. Una descarga de 5MB requiere que TCP "corte" el archivo en miles de fragmentos. Wireshark te advierte que ese paquete es solo un trozo de algo más grande.
